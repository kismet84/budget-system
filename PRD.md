## 1. Executive Summary

我们是**施工单位自营的轻量级预算工具**，目标用户是每天与时间赛跑的预算员（王工）。

**核心痛点**：投标季同时管 2-3 个项目，全靠广联达 + 经验套定额，报价全凭感觉——高了丢单，低了亏损。

**解决方案**：让预算员用**自然语言描述施工场景**，AI 自动推荐最合适的定额子目，同时对比**信息价（官方基准）** 和**市场价（自有数据）**，在 30 分钟内出一份有依据的投标报价。

**数据策略**：所有数据（定额/信息价/市场价）均为**手动维护**——每月从湖北数字造价平台下载 Excel，手工导入系统。系统负责检索、分析、对比，不负责采集。

## 2. Problem Statement

### 2.1 谁有这个痛点？

**王工（Primary Persona）**
- 某施工企业预算负责人，同时管 2-3 个项目投标
- 会用广联达但不精通，操作复杂容易出错
- 投标季压力大：每个项目只有 1-2 天出预算

### 2.2 具体痛点

| 痛点 | 现状 | 业务影响 |
|------|------|---------|
| **定额套用靠经验** | 广联达界面复杂，施工员凭记忆套定额，漏项错项 | 报价偏高丢单 / 偏低亏损 |
| **材料价格无参考** | 信息价靠查文件，市场价靠打电话问供应商 | 材料成本偏差 ±15%，利润被吃掉 |
| **历史项目无法复用** | 每个项目从零开始，重复劳动 | 同类工程无法参照，做完就丢 |
| **报价输出格式乱** | 广联达报表固定，改格式反复调整 | 客户沟通成本高 |

### 2.3 证据

- **调研数据**：湖北数字造价平台有完整的材料价格数据（17城市，1807种材料，54,057条历史价格记录）
- **定额数据**：湖北省2018版园林定额+2024版房建定额（征求意见中），官方免费 Excel 可获取
- **用户行为**：预算员日常工作流程完全依赖 Excel 手工整理（飞书群/微信询价截图）

## 5. Solution Overview

### 5.1 系统定位

**轻量级预算工具**，专注**快速报价估算**和**成本分析**两个高频场景。

**不追求替代广联达**（清单编制不在本期范围），也不做多租户团队协作（纯自用）。

### 5.2 核心功能（MoSCoW）

| 优先级 | 功能 | 描述 |
|--------|------|------|
| **Must** | **AI 定额语义匹配** | 用户描述施工场景 → 自动推荐定额子目（Top3）+ 信息价/市场价对比 |
| **Must** | **数据手动维护** | 定额/信息价/市场价均为 Excel 导入，管理员操作 |
| **Must** | **预算书生成** | 一键输出标准格式预算书（PDF / Excel）|
| **Should** | **历史项目复用** | 类似工程标签化管理，快速复制并调整 |
| **Should** | **成本分析** | 人/材/机费用占比，分部分项费用占比，单方指标 |
| **Could** | **投标策略辅助** | 同类项目中标价对比，报价区间建议 |
| **Won't** | **CAD 工程量识别** | 设计阶段需求，后续再说 |
| **Won't** | **多租户协作** | 自用系统，不需要团队权限管理 |

### 5.3 数据维护流程（手动模式）

```
每月两次（15日/月末）
    │
    ▼
从湖北数字造价平台下载 Excel
bzde.hbcic.net.cn/#/home/dynamicAndPolicy?sysCode=001027001
    │
    ▼
管理员登录系统 → 数据导入模块
    │
    ├── 信息价导入（Excel → 系统）
    ├── 市场价录入（手工 / Excel 批量）
    └── 定额更新（新版发布后导入）
    │
    ▼
数据生效 → 用户使用 AI 匹配 / 价格查询
```

### 5.4 用户流程

**PC 端（主操作）**：
```
登录 → 选择/新建项目 → AI 语义匹配（输入施工描述）
→ 查看推荐定额 Top3 → 人工确认/调整
→ 补充材料价格 → 生成预算书 → 导出 Excel/PDF
```

**移动端（查询 + AI）**：
```
登录 → 输入施工描述（自然语言）
→ 查看 AI 推荐定额（Top3）+ 信息价/市场价对比卡片
→ 复制结果到 PC 端继续操作
```

## 7. User Stories & Requirements

### Epic Hypothesis

> 我们相信，为预算员提供 **AI 定额语义匹配 + 信息价/市场价双轨管理**，能让他们在 30 分钟内完成一份有依据的投标报价，提高报价准确性并减少定额套用错误。我们将通过**单项目预算编制时间 ≤ 30 分钟**来衡量是否成功。

### User Stories

#### US-1：定额数据导入（管理员）

**故事**：作为系统管理员，我需要将官方 Excel 定额数据导入系统，以便后续 AI 匹配和预算计算使用。

**验收标准**：
- [ ] 管理员上传 Excel 文件（园林定额 2018 版），系统解析入库
- [ ] 定额数据按分部工程分类（绿化/园路/园林景观/措施项目）
- [ ] 每个定额子目包含：编号/名称/单位/基价/人工/材料/机械/主材/设备/费用/增值税
- [ ] 定额更新时，支持增量导入（追加新版，保留旧版）

#### US-2：AI 定额语义匹配（核心）

**故事**：作为预算员，我在投标现场用手机描述施工场景，系统自动推荐最合适的定额子目。

**验收标准**：
- [ ] 输入自然语言（如"浇筑 C30 混凝土矩形柱，模板周转5次"），返回 Top3 定额推荐
- [ ] 每条推荐显示：定额编号/名称/综合单价/人材机明细摘要
- [ ] 用户可接受/调整/换算推荐定额
- [ ] AI 匹配响应时间 ≤ 5 秒（移动端）

#### US-3：材料价格管理

**故事**：作为预算员，我需要录入市场价，并与官方信息价进行对比。

**验收标准**：
- [ ] 管理员每月导入信息价 Excel（湖北数字造价平台下载）
- [ ] 用户可录入/编辑市场价（材料名称/规格/供应商/单价/日期）
- [ ] 同一种材料，同时显示信息价和市场价
- [ ] 价格有效期标注（市场价默认30天有效，过期提示更新）

#### US-4：预算书生成

**故事**：作为预算员，我完成定额选取和价格补充后，一键生成标准预算书。

**验收标准**：
- [ ] 预算书包含：封面 + 分部分项清单 + 汇总表
- [ ] 封面信息：工程名称/日期/编制人
- [ ] 支持导出 PDF / Excel
- [ ] 符合湖北省现行计价规范格式

#### US-5：历史项目复用

**故事**：作为预算员，我想复用以前做过的类似项目，快速调整生成新预算。

**验收标准**：
- [ ] 项目建档：名称/类型/面积/中标价/开工日期
- [ ] 支持标签管理（结构类型/地区/年份）
- [ ] 输入新工程信息，推荐最相似的3个历史项目
- [ ] 一键复制历史项目为新项目模板

#### US-6：成本分析

**故事**：作为老板，我想看懂预算书的费用构成，与历史项目对比。

**验收标准**：
- [ ] 显示人/材/机三大费用占比（饼图）
- [ ] 分部分项费用占比（柱状图）
- [ ] 单方指标（元/㎡）：与历史项目对比
- [ ] 不需要操作广联达，在系统内直接查看

### 功能约束与边界情况

| 情况 | 处理方式 |
|------|---------|
| AI 匹配无结果 | 提示"未找到合适定额，请尝试调整描述或手动选择" |
| 市场价未录入 | 仅显示信息价，标注"市场价待补充" |
| Excel 导入格式错误 | 显示具体错误行号，导入中止，不产生脏数据 |
| 定额库为空 | 引导先完成 US-1（定额导入）再使用 AI 匹配 |

## 8. Out of Scope

| 功能 | 原因 |
|------|------|
| **爬虫/自动数据采集** | 改为手动维护（信息价/市场价/定额均从官方 Excel 导入）|
| **多租户/团队权限管理** | 自用系统，单账号即可 |
| **工程量 CAD 识别** | 设计阶段需求，后续版本考虑 |
| **广联达 GBQ 格式兼容** | 输出标准 Excel，客户自行导入广联达 |
| **投标策略 AI** | 需要中标历史数据积累，MVP 阶段不做 |

- **下载地址**：`#/home/dynamicAndPolicy?sysCode=001027001` → 点"查看"→ 点"下载"

- **发布规律**：每月 10-15 日发布上月数据

### A2. 定额数据

- **当前数据**：园林绿化 2018 版（可查询，含人材机明细）
- **2024 版进度**：房建与装饰工程消耗量定额（2024）正在征求意见（2026-03-25），正式发布后需第一时间导入

### A3. 数据入库主键设计建议

```
材料价格表主键：{材料编号}_{城市}_{月份}
```
