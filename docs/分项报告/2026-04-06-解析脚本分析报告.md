## 4. extract_sub_items_from_blocks 分析

### 4.1 逻辑流程

```
1. page.get_text('words') 获取词级坐标
2. 找含字母+数字的词（定额代码）→ x 坐标
3. 计算列边界：code_xs[0]-80, (code_xs[i-1]+code_xs[i])/2, ...
4. item_y 范围：code_y + 50~115（第1行表头）
5. spec_y 范围：code_y + 115~185（第2行表头）
6. 按最近中心分配词汇到各列
7. 组合品名-规格
```

### 4.2 验证结果

| 定额 | 提取结果 | 期望 | 结果 |
|------|---------|------|------|
| A1-17 | `混水砖方柱-普通砖` | `混水砖方柱-普通砖` | ✅ |
| A1-18 | `多孔砖-普通砖` | `混水砖方柱-多孔砖` | ❌ |
| A1-19 | `普通砖` | `多孔砖（多列合并）` | ⚠️ |

### 4.3 残留问题

**问题A — 列分配边界误差**：
- `混水砖方柱` x=272 在 col1/col2 边界（center1=233, center2=307）
- `多孔砖` x=277 同样在边界
- 导致 col2 匹配到"多孔砖"而非"混水砖方柱"

**问题B — spec 行多列重复**：
- `普通砖` 在 col1(x=272)和 col3(x=421)同时出现
- 算法将 col1 分配给 A1-17, col3 分配给 A1-19
- A1-18 没有匹配到 spec（被其他列"抢走"）

**问题C — 3列以上页扩展性**：
- 当 N>3 时，boundaries 计算可能偏差

## 8. 附录：关键代码片段

### 8.1 past_header 逻辑

```python
past_header = False
for i, line in enumerate(lines):
    stripped = fw(line).strip()
    # 跳过"项/目"分隔行
    if re.match(r'^项$|^目$|^\s*项\s*目\s*$', stripped):
        past_header = True
        continue
    # 只在项/目之后提取分项工程
    if not item and past_header:
        m = re.match(r'^[0-9０-９]+[．、.][\u4e00-\u9fff]', stripped)
        if m:
            item = stripped[m.end():].strip()
```

### 8.2 列边界计算

```python
code_xs = [275, 349, 424]  # A1-17/18/19 x 坐标
N = len(code_xs)
boundaries = []
for i in range(N):
    left  = code_xs[0] - 80 if i == 0 else (code_xs[i-1] + code_xs[i]) / 2
    right = code_xs[i] + 150 if i == N - 1 else (code_xs[i] + code_xs[i+1]) / 2
    boundaries.append((left, right))
```
