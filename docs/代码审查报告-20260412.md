### 后端

**1. 两个 declarative_base() 导致 material_prices 表不会被创建**

- **位置**：`models/quota.py` + `models/price.py`
- **问题**：`models/quota.py` 和 `models/price.py` 各自用 `declarative_base()` 创建了不同的 Base。`database.py` 的 `init_db()` 只导入了一个 Base 的 metadata，调用 `create_all()` 时 material_prices 表永远不会被创建。
- **修复方案**：

  Step 1: 修改 `backend/database.py` — 在 `init_db()` 中导入所有模型

  ```python
  def init_db():
      """初始化数据库表"""
      # 导入所有模型以确保 metadata 包含所有表
      from models.quota import Quota, Material      # noqa
      from models.price import MaterialPrice        # noqa

      # 启用 pgvector 扩展（如果不存在）
      with engine.connect() as conn:
          conn.execute(text("CREATE EXTENSION IF NOT EXISTS vector"))
          conn.commit()

      Base.metadata.create_all(bind=engine)
  ```

  Step 2: 修改 `backend/models/quota.py` — 删除本地 `Base`，从 database 导入

  ```python
  # 改前：
  from sqlalchemy.orm import declarative_base
  Base = declarative_base()  # ← 删除这行

  # 改后：
  from database import Base  # ← 添加这行，从统一入口导入
  ```

  Step 3: 修改 `backend/models/price.py`

  ```python
  # 改前：
  from sqlalchemy.orm import declarative_base
  Base = declarative_base()

  # 改后：
  from database import Base  # ← 添加这行
  ```

  **验证方法**：

  ```bash
  # 重启服务后检查表是否全部创建
  psql -d budget_system -c "\dt"

  # 应该看到 quotas, materials, material_prices 三张表
  # 如果之前服务曾启动但 material_prices 缺失，先删除旧表再重启：
  psql -d budget_system -c "DROP TABLE IF EXISTS material_prices CASCADE;"
  pm2 restart budget-api
  ```

**2. Boost 分数覆盖原始语义相似度，阈值过滤失效**

- **位置**：`routers/ai_search.py` 第 246 行
- **问题**：`similarity = min((r.get("similarity") or 0) + boost_amt, 1.0)` 直接修改原始分数。之后 `min_similarity` 阈值过滤的是 Boost 后的分数，而非真实余弦相似度。
- **修复方案**：

  ```python
  # 改前：
  r["similarity"] = min((r.get("similarity") or 0) + boost_amt, 1.0)

  # 改后：保留原始相似度，用单独的 boost_score 字段
  raw_sim = r.get("similarity") or 0
  boost_score = boost_score or 0 + boost_amt  # 新字段
  r["raw_similarity"] = raw_sim
  r["similarity"] = min(raw_sim + boost_score, 1.0)
  ```

  同时修改过滤逻辑使用 `raw_similarity` 而非 `similarity`：

  ```python
  # 改前：
  if (r.get("similarity") or 0) < min_similarity:
      continue

  # 改后：
  if (r.get("raw_similarity") or 0) < min_similarity:
      continue
  ```

  **验证方法**：

  ```bash
  curl -X POST http://localhost:8001/api/v1/ai/search \
    -H "Content-Type: application/json" \
    -d '{"query": "E1-1", "top_k": 5}'

  # 结果中 E1-1 应该排在第 1 位，raw_similarity 应为 1.0
  ```

**3. 中文分词失效，所有 Boost 规则对中文无效**

- **位置**：`routers/ai_search.py` 第 182 行
- **问题**：`field_words = field.split()` 按空格分词，中文文本没有空格，整句作为单个词，所有基于词的 Boost 规则对中文全部失效。
- **修复方案**：

  Step 1: 安装 jieba

  ```bash
  pip install jieba
  ```

  Step 2: 修改 `backend/services/vector_search.py` — 添加分词函数

  ```python
  import jieba
  import re

  def tokenize_chinese(text: str) -> List[str]:
      """
      中文分词 + 过滤停用词 + 过滤单字符
      返回有意义的词列表
      """
      if not text:
          return []
      stop_words = {"的", "了", "和", "与", "或", "及", "在", "于", "为", "以", "等", " ", "　"}
      words = jieba.cut(text)
      return [w for w in words if len(w) >= 2 and w not in stop_words]
  ```

  Step 3: 修改 `search_by_keyword()` — 在 ILIKE 搜索前加分词步骤

  ```python
  # 在 exact quota_id 匹配之后，substring search 之前添加：
  query_words = tokenize_chinese(keyword)
  if query_words:
      # 用 AND 连接多个 ILIKE 条件
      conditions = " OR ".join(
          ["(work_content ILIKE %s OR section ILIKE %s OR quota_id ILIKE %s)"] * len(query_words)
      )
      params = []
      for w in query_words:
          pat = f"%{w}%"
          params.extend([pat, pat, pat])
      params.append(top_k)
      cur.execute(f"SELECT ... FROM quotas WHERE ({conditions}) LIMIT %s", params)
  ```

  **验证方法**：

  ```python
  import jieba
  print(list(jieba.cut("内墙抹灰 20mm厚")))
  # 预期输出：['内墙', '抹灰', '20mm', '厚']
  ```

**6. text_to_vector 同步阻塞事件循环**

- **位置**：`services/vector_search.py` 第 89 行
- **问题**：`urllib.request.urlopen()` 是同步调用，在 async 路由中直接调用会阻塞事件循环。
- **修复方案**：

  ```python
  # 改前：
  response = urllib.request.urlopen(req, timeout=30)

  # 改后：
  import asyncio
  loop = asyncio.get_event_loop()
  response = await loop.run_in_executor(None, lambda: urllib.request.urlopen(req, timeout=30))
  ```

## Error Boundary 详细方案

Step 1: 新建 `frontend/src/components/ErrorBoundary.tsx`

```tsx
import React, { Component, ErrorInfo, ReactNode } from 'react'

interface Props { children: ReactNode; fallback?: ReactNode }
interface State { hasError: boolean; error: Error | null }

export default class ErrorBoundary extends Component<Props, State> {
  constructor(props: Props) {
    super(props)
    this.state = { hasError: false, error: null }
  }

  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error }
  }

  componentDidCatch(error: Error, errorInfo: ErrorInfo) {
    console.error('[ErrorBoundary]', error, errorInfo)
  }

  render() {
    if (this.state.hasError) {
      if (this.props.fallback) return this.props.fallback
      return (
        <div className="min-h-screen bg-slate-900 flex items-center justify-center">
          <div className="text-center">
            <div className="text-red-400 text-lg font-medium mb-2">页面发生错误</div>
            <div className="text-slate-400 text-sm">{this.state.error?.message}</div>
            <button
              onClick={() => this.setState({ hasError: false, error: null })}
              className="mt-4 px-4 py-2 bg-slate-700 text-white rounded-lg text-sm"
            >
              重试
            </button>
          </div>
        </div>
      )
    }
    return this.props.children
  }
}
```

Step 2: 修改 `frontend/src/main.tsx`

```tsx
import ErrorBoundary from './components/ErrorBoundary'

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <ErrorBoundary>
      <App />
    </ErrorBoundary>
  </React.StrictMode>,
)
```
